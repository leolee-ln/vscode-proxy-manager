# 更新日志

所有重要的项目变更都会记录在此文件中。

本文件格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)，
项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [1.0.2] - 2025-12-15

### 新增功能
- 新增配置变化检测功能，在执行操作前自动检测配置是否会发生变化
- 添加 `check_proxy_changes()` 方法用于预检测配置变化
- 支持在配置显示时自动隐藏代理密码信息（显示为 `****`）

### 改进
- 优化用户交互流程：先检测变化 → 显示变化内容 → 询问确认 → 执行操作
- 当配置无变化时直接退出，不再询问用户确认
- 仅在配置实际发生变化时才生成备份文件，避免产生不必要的备份
- 启用代理时，如果配置未发生变化，显示"代理配置已是最新状态"并退出
- 禁用代理时，如果已处于禁用状态，显示"代理配置已是禁用状态"并退出
- 配置变化信息只显示一次，提升用户体验

### 优化
- 改进备份逻辑：从"操作前备份"改为"确认变化后备份"
- 减少不必要的文件 I/O 操作
- 优化日志输出，提供更清晰的状态反馈

## [1.0.1] - 2025-12-15

### 修复
- 修复 vscode-proxy.sh 无法正确读取配置文件中 Python 解释器路径的问题
- 修复当配置文件不存在时 grep 命令报错的问题
- 改进 YAML 配置解析逻辑，正确提取 `interpreter:` 字段的值

### 改进
- 增强配置文件读取的健壮性
- 添加配置文件存在性检查
- 优化引号和换行符处理

## [1.0.0] - 2025-12-15

- VSCode Proxy Manager 首次发布

- 自动备份功能，支持版本控制
- YAML 配置文件支持

- 当前代理配置状态显示

- 自定义代理主机和端口
- 自动配置文件备份（最多保留 5 个版本）
- 用户友好的命令行界面
- 灵活的 YAML 配置
- 多种命令别名（enable/on, disable/off）
- 智能配置文件查找（系统 → 项目 → 默认）
